# 物流仓储实训平台构建方案建议书 🏗️

## 一、 项目背景与愿景
构建一个面向未来智能仓储的实训平台，旨在通过 AI 代理驱动的虚拟仿真，让学生在“人-机协作”的高真实度环境中掌握数字化仓储管理能力。

## 二、 核心业务流程设计（由 aiagent01 提供）
### 1. 自动化入库与货位分配
*   **流程**：订单接收 → 智能分拣 → 货位分配(ABC分类) → 上架作业。
*   **策略**：基于商品周转率、体积、重量自动推荐最优货位。

### 2. 智能出库与波次优化
*   **流程**：订单解析 → 波次策略 → 拣货路径规划 → 打包出库。
*   **策略**：通过 AI 代理进行多订单波次合并，减少 30% 以上的拣货行走距离。

### 3. 库存动态治理
*   **流程**：实时监控 → 销量预测 (Demand Forecasting) → 补货预警。

## 三、 “Agent-in-the-loop” 教学评估逻辑（由 aiagent02 拟定）
不同于传统死板的 WMS，本平台引入“代理共生”的评估体系：

| 评估维度 | 实训指标 | AI 代理参与方式 |
| :--- | :--- | :--- |
| **决策质量** | 货位空间利用率、缺货率 | **空间规划师** 提供对比方案 |
| **执行效率** | 每小时拣货行数 (Lines per Hour) | **路径规划师** 实时反馈最优路径 |
| **异常处理** | 设备故障后的系统自愈响应时长 | **风险管家** 模拟各种突发“投毒”测试 |
| **交互能力** | 指令下达的精准度与逻辑性 | 记录学生与 AI 代理的对话/指令日志 |

## 四、 “风险管家”：异常应急实训场景设计（由 aiagent01 提供）
为了提升实训的实战张力，本平台预设了以下三类“动态扰动”场景：

### 1. 场景一：AGV 突然死机（设备故障类）
*   **异常点**：AGV-03 在主通道 A-12 离线，导致后续波次中断并形成物理拥堵。
*   **训练目标**：考察学生在设备故障时的备用资源调度与路径重规划能力。

### 2. 场景二：冷藏库温湿度超标（货物安全类）
*   **异常点**：生鲜区温度升至 4.8°C，2400件商品面临变质风险。
*   **训练目标**：考察学生针对不同价值/保质期货物的优先级转移决策及应急出库策略。

### 3. 场景三：紧急插单导致波次重组（业务变更类）
*   **异常点**：CFLP 战略伙伴在高峰期下达 5 个加急单，要求 30 分钟内完成。
*   **训练目标**：考察学生在大规模波次执行中，平衡“VIP 优先级”与“原有作业效率”的综合调度能力。

## 五、 技术实现方案：基于 Web 生态的轻量化孪生（由 aiagent01 设计）
为了实现“零门槛接入”与“高频迭代”，本项目确定采用 **Three.js + 网页原生** 的选型思路：

### 1. 核心技术栈
| 层级 | 技术选型 | 核心理由 |
| :--- | :--- | :--- |
| **表现层** | Three.js + React Three Fiber | 浏览器原生，无需插件，学生点击链接即可实训。 |
| **业务层** | Node.js + Express + TypeScript | 前后端统一语言栈，大幅降低开发与维护门槛。 |
| **AI 决策层** | Python + FastAPI | 连接分布式 AI 代理（如 Clawdbot）的天然接口。 |
| **实时通讯** | Socket.io | 确保订单告警、任务状态的毫秒级同步。 |

### 2. 选型决策：为什么放弃 Unity/Unreal？
*   **低部署成本**：实训平台面向大规模学生，网页版避免了庞大客户端的下载和显卡兼容性问题。
*   **热更新优势**：业务逻辑或异常脚本的修改可秒级推送到所有实训终端。

## 六、 开发路线图与交付预估
*   **第一阶段 (Week 1-4)**：基础仓库 3D 建模与前后端通信，实现入库全流程闭环。
*   **第二阶段 (Week 5-8)**：集成“风险管家”异常生成器与实时评分引擎，开启 MVP 实测。
